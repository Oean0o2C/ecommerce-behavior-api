# Frontend

Streamlit 前端应用代码目录，提供交互式数据可视化界面。

## 技术栈

- **Python**: 3.11+
- **Streamlit**: 1.32+ - 快速构建数据应用的 Python 库
- **Plotly**: 交互式图表库
- **Pandas**: 数据处理与分析
- **Requests**: HTTP 客户端

## 项目结构

```
frontend/
├── app.py              # Streamlit 应用主入口
├── requirements.txt    # 依赖包列表
└── README.md          # 本文件
```

## 功能特性

- **销售概览仪表盘**：展示 GMV、订单数、UV、转化率等核心 KPI
- **趋势分析**：支持按日/周/月查看销售趋势
- **品类分析**：展示各品类销售额占比和表现
- **用户行为漏斗**：可视化浏览→加购→购买的转化漏斗，显示每步转化率
- **热销商品**：展示按销售额/浏览量/加购量排序的热销商品
- **时间范围选择**：支持最近7天/30天/自定义时间范围
- **数据导出**：支持 CSV 和 Excel 格式导出所有数据
- **API 配置**：侧边栏可配置后端 API 地址，支持测试连接
- **数据缓存**：API 响应缓存5分钟，提升性能

## 本地开发

### 环境要求

- Python 3.11+
- 后端 API 服务已启动（默认 http://localhost:8000）

### 进入虚拟环境

**Windows:**
```bash
# 进入项目根目录
cd d:\Cources\@za\项目1

# 激活虚拟环境
.venv\Scripts\activate
```

**macOS/Linux:**
```bash
# 进入项目根目录
cd /path/to/项目1

# 激活虚拟环境
source .venv/bin/activate
```

激活成功后，命令行提示符前会显示 `(.venv)`。

### 安装依赖

**所有依赖已在项目根目录的 `requirements.txt` 中统一管理。**

```bash
# 确保已在虚拟环境中，然后在项目根目录安装依赖
cd d:\Cources\@za\项目1
pip install -r requirements.txt
```

**注意**：不需要单独安装 frontend 依赖，根目录的 `requirements.txt` 已包含所有前后端依赖。

### 启动应用

**重要：确保已激活虚拟环境（见上文），然后进入 frontend 目录启动应用：**

```bash
# 进入 frontend 目录
cd frontend

# 启动 Streamlit 应用
streamlit run app.py
```

应用启动后，自动在浏览器中打开 http://localhost:8501

## 配置说明

### API 配置

在侧边栏的 "⚙️ API 配置" 区域可以：

1. **设置 API Base URL**：
   - **本地开发**：`http://localhost:8000`
   - **生产环境**：`https://your-backend-url.railway.app`

2. **测试连接**：
   - 点击 "🔄 测试连接" 按钮
   - 验证后端服务是否可访问
   - 显示连接成功或失败状态

### 时间范围选择

在侧边栏的 "📅 时间范围" 区域可以：

- **快速选择**：
  - 最近 7 天
  - 最近 30 天
  
- **自定义范围**：
  - 选择开始日期
  - 选择结束日期

### 数据导出

在侧边栏的 "📥 数据导出" 区域可以选择导出格式：

- **CSV**：所有数据都支持 CSV 格式导出
- **Excel**：品类分析支持 Excel 格式导出（需要安装 openpyxl）

每个数据模块都有独立的导出按钮，可以导出该模块的数据。

## 生产部署

**生产环境**托管在 **Streamlit Cloud**（与根目录 `README.md` 部署架构一致）。

### Streamlit Cloud 部署步骤

1. 登录 [Streamlit Cloud](https://streamlit.io/cloud)
2. 点击 "New app"
3. 选择 GitHub 仓库和分支
4. 指定主文件路径：`frontend/app.py`
5. 配置环境变量（如需要）
6. 点击 Deploy

### 环境变量配置

在 Streamlit Cloud 的 Secrets 中配置：

```toml
[api]
base_url = "https://your-backend-url.railway.app"
```

## 性能优化

- **数据缓存**：使用 `@st.cache_data(ttl=300)` 缓存 API 响应，5分钟内重复请求直接返回缓存数据
- **超时控制**：API 请求设置 10 秒超时，避免长时间等待
- **加载状态**：使用 `st.spinner` 显示加载动画，提升用户体验
- **图表优化**：使用 Plotly 的 `use_container_width=True` 自适应容器宽度
- **避免重渲染**：使用 `session_state` 管理状态，避免不必要的页面刷新

## 注意事项

1. **API 连接**：确保后端服务可访问，否则页面会显示错误信息
2. **数据量**：大数据集查询可能需要几秒时间，建议添加加载状态
3. **浏览器兼容性**：推荐使用 Chrome 或 Edge 浏览器
4. **移动端**：Streamlit 应用支持移动端访问，但图表显示可能不够理想

## 故障排查

### 页面显示 "Connection error"
- 检查 API Base URL 是否正确
- 确认后端服务是否运行
- 检查网络连接

### 图表不显示
- 检查浏览器控制台是否有 JavaScript 错误
- 确认数据格式是否正确
- 尝试刷新页面

### 数据加载缓慢
- 检查后端 API 响应时间
- 考虑缩小时间范围
- 查看是否有网络延迟
